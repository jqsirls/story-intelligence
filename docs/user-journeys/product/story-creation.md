# Story Creation Journeys

**Last Updated:** December 2025  
**Audience:** Product, Design, Marketing, Support

## Overview

This document walks through how users create stories on Storytailor, from first story to advanced creation with characters and customization.

---

## Journey 1: First Story Creation

**New user creating their first story.**

### What Happens

**Step 1: User Logs In**
- Emma (age 8, with parent approval) logs into Storytailor.com
- Sees homepage: "Ready to create your first story?"
- Button: "Create New Story"

**Step 2: Start Story Creation**
- Emma clicks "Create New Story"
- System asks: "What type of story would you like?"
  - Adventure
  - Bedtime
  - Educational
  - Funny
  - Magical
  - Custom
- Emma selects: "Adventure"

**Step 3: Choose or Create Character**
- System asks: "Who is the hero of your adventure?"
- Options:
  - "Create a new character"
  - "Use a character from the library" (if any exist)
- Emma clicks: "Create a new character"

**Step 4: Character Creation**
- Emma describes her character:
  - Name: "Brave the Dragon"
  - Traits: "Brave, kind, loves flying"
  - Appearance: "Purple with gold scales"
  - Age: "Young dragon (like 8 years old)"
- System shows preview: Character illustration generated
- Emma confirms: "Yes, that's Brave!"

**Step 5: Story Details**
- System asks story questions:
  - "Where does the adventure take place?" → Emma: "In the clouds"
  - "What is Brave trying to do?" → Emma: "Find a lost treasure"
  - "What makes this treasure special?" → Emma: "It grants wishes"
  - "Who else is in the story?" → Emma: "A wise old owl friend"

**Step 6: Story Generated**
- System creates personalized story
- Loading message: "Creating your adventure..."
- Takes 15-30 seconds
- Story includes:
  - Brave the Dragon as hero
  - Adventure in the clouds
  - Age-appropriate length and words (for 8-year-old)
  - Positive themes and character growth
  - Wish-granting treasure quest

**Step 7: Story Preview**
- Emma sees her complete story
- Can read it immediately
- Options:
  - "Read Story" (text version)
  - "Listen" (audio version, if available)
  - "Save to Library"
  - "Create Another Story"
  - "Edit Story" (make changes)

**Step 8: Story Saved**
- Emma clicks "Save to Library"
- Story saved to "Emma's Stories"
- Can access anytime
- Can share with family (if parent approves)

### What Data Was Collected

During this journey, system saved:
- Character: "Brave the Dragon" with traits
- Story: Complete story text
- Preferences: Emma likes adventure stories, dragon characters

### Key Points

- Simple, guided process
- Minimal data collection
- Age-appropriate content
- User controls what to save
- Can create multiple stories

## Journey 2: Quick Story with Existing Character

**Experienced user creating story quickly.**

### What Happens

**Step 1: User Logs In**
- Emma returns to Storytailor
- Sees her character library with "Brave the Dragon"

**Step 2: Quick Create**
- Emma says or types: "Create a bedtime story with Brave"
- Or clicks "Create Story" and selects "Brave" from character list

**Step 3: Story Type Auto-Detected**
- System recognizes "bedtime story" request
- Automatically selects calming theme
- Uses existing character Brave

**Step 4: Quick Questions**
- "Where is Brave tonight?" → Emma: "In a cozy cloud castle"
- "What happens?" → Emma: "Gets ready for sleep"

**Step 5: Bedtime Story Generated**
- Story created optimized for bedtime:
  - Calming tone
  - Shorter length
  - Peaceful ending
  - Brave goes to sleep at end

**Step 6: Story Saved Automatically**
- Auto-saved to Emma's library
- Titled: "Brave's Bedtime in the Clouds"
- Ready to read immediately

### Key Points

- Faster flow for returning users
- Reuses existing characters
- Context-aware (bedtime vs. adventure)
- Auto-save reduces friction
- Sub-5 minute creation

## Journey 3: Story Creation with Multiple Characters

**Creating a story with several characters.**

### What Happens

**Step 1: Start New Story**
- User selects "Create New Story"
- Chooses "Adventure"

**Step 2: Select Multiple Characters**
- System asks: "Who are the heroes?"
- User can select multiple characters:
  - "Brave the Dragon" (existing)
  - "Luna the Mouse" (existing)
  - "+ Add new character"
- User adds both existing characters

**Step 3: Character Relationships**
- System asks: "How do Brave and Luna know each other?"
- User: "Best friends who go on adventures together"

**Step 4: Story Details**
- Where: "Enchanted forest"
- What happens: "They search for magical crystals"
- Challenge: "Must work together to solve puzzles"

**Step 5: Story Generated**
- Story features both characters
- Shows their friendship
- Includes teamwork and puzzle-solving
- Age-appropriate challenges

**Step 6: Story Saved**
- Saved with both character associations
- Both characters' story lists updated
- Available in user's library

### Key Points

- Supports multiple characters
- Character relationships matter
- Story adapts to character dynamics
- All characters linked to story

## Journey 4: Editing an Existing Story

**Making changes to a saved story.**

### What Happens

**Step 1: Open Story**
- User goes to library
- Selects story: "Brave's Cloud Adventure"
- Clicks "Edit Story"

**Step 2: Edit Options**
- Change story details
- Add/remove characters
- Adjust story length
- Change story type/theme
- Regenerate parts

**Step 3: Make Changes**
- User changes: "Add a villain dragon"
- System: "Tell me about the villain"
- User describes villain

**Step 4: Story Regenerated**
- System updates story with villain
- Keeps original plot intact
- Adds new character naturally
- Maintains age-appropriateness

**Step 5: Save Changes**
- User reviews updated story
- Clicks "Save Changes"
- Original story replaced (or saved as new version)

### Key Points

- Stories are not locked after creation
- Can edit and regenerate
- Changes feel natural (not forced)
- User controls final version

## Journey 5: Story Types and Themes

**Different story types available.**

### Available Story Types

**Adventure:**
- Action-packed quests
- Challenges and triumphs
- Hero's journey structure
- Age-appropriate excitement

**Bedtime:**
- Calming, peaceful stories
- Shorter length
- Ends with sleep/rest
- Soothing tone

**Educational:**
- Learning woven into narrative
- Age-appropriate concepts
- Problem-solving
- Character learns something

**Funny:**
- Humorous situations
- Silly characters
- Playful tone
- Laughter and joy

**Magical:**
- Fantasy elements
- Imagination and wonder
- Magical powers or items
- Dream-like quality

**Custom:**
- User defines all parameters
- Mix of themes
- Specific topics
- Personalized to request

### Key Points

- Multiple story types available
- Each optimized for purpose
- Age-appropriate for each type
- Can create custom stories

## Journey 6: Story with Therapeutic Elements

**Creating stories for emotional support (Story Intelligence feature).**

**Note:** This feature becomes available when Story Intelligence is integrated (future).

### What Happens

**Step 1: Emotional Check-In**
- System asks: "How are you feeling today?"
- Emma: "A little worried about school tomorrow"

**Step 2: Story Type Selected**
- System suggests: "Would you like a story about feeling brave?"
- Emma: "Yes"

**Step 3: Therapeutic Story Created**
- Story about Brave facing a challenge
- Character feels worried at first
- Character finds courage
- Positive resolution
- Validates Emma's feelings

**Step 4: Story Supports Emma**
- Story acknowledges worry is normal
- Shows character overcoming worry
- Provides coping strategies naturally
- Age-appropriate emotional support

### Key Points

- Stories can support emotional needs
- Not therapy, but therapeutic
- Age-appropriate emotional content
- Validates feelings while providing hope
