# Parental Controls Journeys

**Last Updated:** December 2025  
**Audience:** Product, Design, Marketing, Support

## Overview

This document walks through all parent-specific features for managing children's accounts, exercising parental rights, and maintaining privacy controls.

---

## Journey 1: Checking Consent Status

**Parent checking if they've approved their child's account.**

### What Happens

**Step 1: Parent Logs Into Storytailor.com**
- Mom logs into her Storytailor account
- Goes to Account Settings → Manage Children's Accounts/Profiles

**Step 2: View Consent Information**
- Sees Emma's account with full details:
  - Consent status: "Approved"
  - Date approved: "January 15, 2025"
  - Data collection: "Active"
  - Stories created: 5
  - Last active: "Today at 3:42 PM"

**Step 3: Review Options**
- Can click "View Details" to see:
  - Full consent history
  - What data has been collected
  - When consent was given
  - Options to revoke or manage settings

### Alternative Method: Email Inquiry

If parent doesn't have account access:
- Email privacy@storytailor.com
- We verify identity and send status
- Response within 24-48 hours

### Key Points

- Instant access to consent status
- Complete transparency
- Can review anytime
- See child's activity level

## Journey 2: Revoking Consent

**Parent stopping their child's access to Storytailor.**

### What Happens

**Step 1: Parent Logs Into Storytailor.com**
- Dad logs into his account
- Goes to Account Settings → Manage Children's Accounts
- Sees Lily's account

**Step 2: Access Parental Controls**
- Clicks on Lily's account
- Sees "Parental Controls" section:
  - Current consent status: "Approved"
  - Approval date: "January 15, 2025"
  - Data collection: "Active"
  - [Revoke Consent] button

**Step 3: Revoke Consent**
- Dad clicks "Revoke Consent"
- System asks: "Are you sure? This will stop Lily from using Storytailor."
- Dad confirms: "Yes, Revoke Consent"

**Step 4: Consent Revoked Immediately**
- Consent status changes to "Revoked"
- Data collection stops instantly
- Lily is logged out automatically
- Timestamp recorded

**Step 5: Confirmations**
- Dad sees on-screen confirmation
- Dad receives email confirmation
- Lily receives email: "Your account access has been revoked by your parent"

**Step 6: Optional: Delete Data**
- Screen shows: "Would you like to delete Lily's data?"
- Options:
  - "Delete All Data"
  - "Keep Data (in case you change your mind)"
- Dad chooses action

### Alternative Method: Email Request

If parent doesn't have account access:
- Email privacy@storytailor.com
- We verify identity and process
- Takes 24-48 hours vs. instant

### Key Points

- Instant revocation through website
- Clear confirmation process
- Both parent and child notified
- Can keep or delete data
- Reversible (parent can re-approve later)

## Journey 4: Deleting Child's Data

**Parent permanently removing all child data.**

### What Happens

**Step 1: Parent Logs Into Storytailor.com**
- Dad logs into his account
- Goes to Account Settings → Manage Children's Accounts
- Clicks on Lily's account

**Step 2: Access Deletion Option**
- Dad clicks "Privacy & Data"
- Scrolls to "Delete All Data" section
- Clicks "Delete Lily's Data"

**Step 3: Confirmation Screen**
- System shows what will be deleted:
  - All stories (5 stories)
  - All characters (3 characters)
  - All account information
  - All consent records
  - (Future: Voice conversations, emotional data)
- Warning: "This cannot be undone"
- Note: "Audit logs will be anonymized but kept for legal compliance"

**Step 4: Dad Confirms Deletion**
- Dad enters his password (security check)
- Checks box: "I understand this cannot be undone"
- Clicks "Delete All Data"

**Step 5: Data Deleted Immediately**
- All of Lily's data permanently deleted
- Cannot be recovered
- Audit logs anonymized (personal info removed)
- Deletion timestamp recorded

**Step 6: Confirmations**
- Screen shows: "Lily's data has been deleted"
- Dad receives email with:
  - Deletion timestamp
  - What was deleted
  - What was anonymized
  - Confirmation number
- Lily receives email: "Your account has been deleted"

### Alternative Method: Email Request

If parent doesn't have account access:
- Email privacy@storytailor.com
- We verify identity (may require call/video for security)
- Process within 24-48 hours

### Key Points

- Complete deletion available
- Extra security (password confirmation)
- Cannot recover after deletion
- Audit logs anonymized (legal requirement)
- Both parent and child notified

## Journey 5: Exporting Child's Data

**Parent getting a complete copy of child's information.**

### What Happens

**Step 1: Parent Logs Into Storytailor.com**
- Mom logs into her account
- Goes to Account Settings → Manage Children's Accounts
- Clicks on Emma's account

**Step 2: Request Data Export**
- Mom clicks "Privacy & Data"
- Clicks "Download All Data"
- System asks: "This will create a complete copy of Emma's data"
- Mom confirms: "Yes, Download"

**Step 3: System Prepares Export**
- Loading message: "Preparing Emma's data..."
- System collects:
  - All stories (5 stories with full text)
  - All characters (3 characters with traits)
  - Account information
  - Consent history
  - Safety records (if any)
  - (Future: Voice conversations, emotional data)

**Step 4: Download Ready**
- Screen shows: "Your download is ready!"
- File size: 2.5 MB
- Format: JSON (readable in any text editor)
- [Download Now] button

**Step 5: Mom Downloads File**
- Clicks "Download Now"
- File downloads: storytailor-emma-export-2025-01-15.json
- Can open in text editor or import to other tools
- Complete copy of all Emma's data

**Step 6: Can Export Anytime**
- Mom can request new export anytime
- Always gets current/latest data
- No limit on export requests

### Alternative Method: Email Request

If parent doesn't have account access:
- Email privacy@storytailor.com
- We verify identity and create export
- Secure download link sent (expires in 7 days)
- Provided within 30 days

### Key Points

- Instant data export from website
- Complete data in portable format
- Can export repeatedly
- No expiration on downloaded files
- Supports data portability rights

## Journey 6: Managing Child's Account Settings

**Parent updating settings for child's account.**

### What Happens

- Parent logs in
- Goes to child's account/profile
- Clicks "Account Settings"

**Step 2: Available Settings**

- Child sees updated settings next login

### Key Points

- Changes take effect immediately
- Age-based content adjusts automatically
- Parent has full control

## Journey 7: Profile and Email Updates

**Updating parent's email address for consent communications.**

### What Happens

**Step 1: Parent Wants to Update Email**
- Mom's email changing from old@example.com to new@example.com
- Logs into Storytailor.com
- Goes to "Account Settings"

**Step 2: Update Email**
- Clicks "Update Email"
- Enters new email: new@example.com
- System asks for password (security check)
- Confirms update

**Step 3: Verification Email Sent**
- Email sent to new@example.com
- Subject: "Verify your new email for Emma's account"
- Contains verification link

**Step 4: Mom Verifies New Email**
- Mom clicks verification link
- New email confirmed
- Old email receives notification: "Your email was changed"

**Step 5: All Communications Updated**
- Consent communications go to new email
- Data access requests go to new email
- Safety notifications go to new email
- Child's account settings updated

### Key Points

- Verified email changes
- Security check (password required)
- Old email notified of change
- Seamless update across all systems
